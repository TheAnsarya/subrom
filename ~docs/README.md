# Subrom Development Documentation

This folder contains all development documentation for the Subrom ROM Manager project.

## Folder Structure

```
~docs/
├── README.md                    # This file
├── session-logs/               # AI session logs
├── chat-logs/                  # AI chat conversation logs
├── plans/                      # Project planning documents
│   ├── roadmap.md             # Overall project roadmap
│   ├── architecture.md        # System architecture
│   ├── ui-plans.md            # UI design and plans
│   ├── api-design.md          # API design documentation
│   └── features/              # Feature-specific plans
├── issues/                     # GitHub issue templates and tracking
│   ├── epics.md               # Epic tracking document
│   └── templates/             # Issue templates
└── research/                   # Research and references
	├── dat-formats.md         # DAT file format documentation
	├── providers.md           # ROM set providers (TOSEC, No-Intro, etc.)
	└── competitors.md         # Analysis of RomVault, ClrMame, etc.
```

## Documentation Standards

1. All session logs follow format: `YYYY-MM-DD-session-NN.md`
2. All chat logs follow format: `YYYY-MM-DD-chat-NN.md`
3. Use tabs for indentation (per project standards)
4. Link all issues to parent epics
5. Reference GitHub issues in commit messages
